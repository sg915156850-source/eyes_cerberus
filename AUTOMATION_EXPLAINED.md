## ❓ ВОПРОС 1: Может ли System Eyes автоматически изолировать вредоносы?

### ✅ ДА - ТЕПЕРЬ МОЖЕТ!

**Скрипт:** `/root/eyes_cerberus/auto_containment.sh`

### Что делает автоматически:

| Действие | Автоматически? | Описание |
|----------|----------------|----------|
| Обнаружение NC к C2 | ✅ Да | Мониторит подключения к 107.175.89.136, 87.121.84.56 |
| Обнаружение майнеров | ✅ Да | Ищет xmrig, minerd, nanopool, и т.д. |
| Высокий CPU | ⚠️ Частично | Предупреждает, убивает если хэш известен |
| Kill процесса | ✅ Да | `kill -9` + дети |
| Копирование в изоляцию | ✅ Да | В `/var/malware_containment/malware/` |
| Блокировка файла | ✅ Да | `chmod 000` (НЕ удаляет!) |
| Блокировка сети | ✅ Да | `iptables` правила |
| Логирование | ✅ Да | Всё в `/root/eyes_cerberus/evidence/` |

### Как работает:

```
┌─────────────────────────────────────────────────────────────────┐
│              AUTO-CONTAINMENT ЦИКЛ (10 сек)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. DETECT NC → 2. KILL → 3. ISOLATE → 4. BLOCK              │
│       ↓                                                        │
│  5. DETECT MINERS → 2. KILL → 3. ISOLATE                     │
│       ↓                                                        │
│  6. DETECT HIGH CPU → CHECK HASH → KILL если известен        │
│       ↓                                                        │
│  7. DETECT MALWARE FILES → BLOCK если не заблокирован        │
│       ↓                                                        │
│  [SLEEP 10 SEC] → ПОВТОРИТЬ                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Запуск:

```bash
# Запустить автоматически
./auto_containment.sh

# Или в фоне
nohup ./auto_containment.sh > auto_containment.out 2>&1 &

# Проверить статус
pgrep -f "auto_containment.sh" && echo "РАБОТАЕТ" || echo "НЕ РАБОТАЕТ"

# Остановить
pkill -f "auto_containment.sh"
```

## ❓ ВОПРОС 3: Убивает ли он процессы майнеров/вредоносов?

### ✅ ДА - ТЕПЕРЬ УБИВАЕТ!

**auto_containment.sh** автоматически убивает:

| Тип | Паттерны | Действие |
|-----|----------|----------|
| **NC к C2** | `nc 107.175.89.136`, `nc 87.121.84.56` | ✅ Kill + Isolate |
| **NC к порту 9009** | `nc .* 9009` | ✅ Kill + Block port |
| **Майнеры** | `xmrig`, `minerd`, `cpuminer`, `cryptonight` | ✅ Kill + Isolate |
| **Майнинг-пулы** | `nanopool.org`, `nicehash.com`, `minergate.com` | ✅ Kill connection |
| **Известные хэши** | MD5 из `known_malware_hashes.txt` | ✅ Kill + Isolate |
| **Файлы по путям** | `/let`, `/var/let`, и т.д. | ✅ Block (chmod 000) |

### Пример логики:

```bash
# Обнаружил NC к C2
ps aux | grep "nc 107.175.89.136"
# → Kill -9 PID
# → Копируешь файл в /var/malware_containment/
# → chmod 000 оригинал
# → iptables -A OUTPUT -d 107.175.89.136 -j DROP

# Обнаружил майнер
ps aux | grep "xmrig"
# → Kill -9 PID
# → Копируешь /proc/PID/exe в изоляцию
# → Логируешь в evidence
```
